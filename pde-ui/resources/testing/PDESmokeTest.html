Short Smoke Test - PDE

Last updated May 25th 2001

Note: PDE is based on Java Tooling. There is little point in running PDE smoke test if Java Tooling fails the basic test points.

Setup

  • Open Preferences->Plug-in Development Environment
  • Ensure that Eclipse Home is set and that there is a number of checked plug-ins in the list
  • Set the workspace location to a previously created (empty) directory
  • Switch to PDE perspective (Perspective->Open->Other...->Plug-in Perspective)
  • Ensure that JDK rt.jar and source zip are set in Java Tooling. Default VM that supports JDI must also be set.

Plug-in Project Creation

  • Bring up "Plug-in Project" wizard.
  • Type in "com.example.xyz" for the project name. Press Next.
  • Verify that offered library name is "xyz.jar" and source folder is "XYZ Plugin". Press Next.
  • Switch to using plug-in content template wizards and select the default wizard from the list. Press Next.
  • Accept the offered defaults in the wizard and press 'Finish'.
  • Verify that the new project is created with "XYZ Plugin" folder that contains XYZPlugin.java class.
  • Verify that build.properties is set with the following variable set: 
    source.xyz.jar = XYZ Plugin
  • Verify that bin directory is created and that it contains XYZPlugin.class (result of the build).
  • Plug-in manifest editor should open up.

Plug-in Manifest Editing

  • Overview page should show no alerts or action items. There should be three required plug-ins (Core Resources, Core Runtime and Workbench UI).
  • Dependencies page should show these three external plug-ins checked and the graph should show the visually.
  • Runtime page should have one library (xyz.jar) and when this library is selected, XYZ Plugin folder should show up in the source folder list.
  • Extensions and Extension Points page should look normal, but must be empty.
  • Source page should show the initial XML content with color coding.

Adding an extension

  • Open Extension Points page. Select "New" button and pick "Schema-based Extension" wizard from the list. Press "Next".
  • Select "Action Sets" extension point from the list. Press "Finish".
  • Select "Action Sets" object and select "New->actionSet" from the pop-up menu.
  • Change the name in the property sheet to "Smoke Action Set".
  • Select "New->menu" from the pop-up menu. Change the label of the menu into "Smoke Menu". Change the Id of the menu to "smokeMenu".
  • Select "New->separator" on the menu object. Change the name of the separator to "smokeGroup".
  • Select "New->action" from the pop-up menu while menu is selected. 
  • Select the new action and change its label to "Smoke Action", its menubarPath to "smokeMenu/smokeGroup".
  • Select the "class" property and bring up the dialog. Choose to create a new class. Browse for the source container and pick "XYZ Plugin" folder. Change the name of the class to "SmokeAction". Press Finish.
  • When Java editor opens, add "System.out.println("Smoke test works!");" in the "run" method. Save and close.
  • Save the manifest editor.

Running

  • Select the project resource and press "Run" button. Another Eclipse should show up (if you forgot to set the default VM, bring up the properties for the project and set the VM to use).
  • Open "Perspective->Customize->Other...". Verify that "Smoke Action Set" is available. Check it.
  • Verify that "Smoke Menu" appears on the menu bar. 
  • Verify that the menu contains "Smoke Action" item. Press it.
  • Verify that Console view shows "Smoke test works!".

Debugging

  • Place a breakpoint in the constructor of XYZPlugin.java.
  • Place a breakpoint in 'run' method of SmokeAction.java.
  • Select the project and press Debug tool bar button.
  • When the Eclipse window shows up, select "Smoke Menu->Smoke Action". Wait for the debugger to stop.
  • Verify that the debugger stopped in the constructor of XYZPlugin.java. Continue.
  • Verify that the debugger stopped in 'run' method of SmokeAction.java. Continue.
  • Verify that "Smoke test works!" shows up on the Console.

Adding an extension point

  • Open Extension Points page. Select "New" to open the wizard.
  • Type "parsers" for the extension point Id, "Sample Parsers" for the name. Press "Finish".
  • Verify that Schema editor opened up with "extension" element and three attributes (id, name and point).
  • Verify that there is four documentation topics (Overview, Examples, API Information and Supplied Implementation).
  • Verify that Source page shows XML Schema source with color coding.
  • Close the schema and verify that extension point is showing in the manifest editor.
  • Save and switch to the Source view. 
  • Verify that extension point markup has been added to the source.
